配置用户名和email

$ git config --global user.name "Your Name"

$ git config --global user.email "email@example.com"

选择一个目录，创建版本库

$ git init

Initialized empty Git repository in /Users/michael/learngit/.git/

创建文件index.txt编辑点内容，在添加到仓库

$ git add .

把文件提交到仓库，使用git commit -m 后面加上说明

$ git commit -m "my frist file"

假设我们要改动index.txt文件， 改动后要再次提交，提交之前来看看仓库当前的状态，使用 git status

$ git status

On branch master
Changes not staged for commit:
(use "git add ..." to update what will be committed)
(use "git checkout -- ..." to discard changes in working directory)
modified: index.txt
no changes added to commit (use "git add" and/or "git commit -a")

上面命令告诉我们，index.txt文件被修改了，但是还没有准备提交的修改

要查看具体修改了什么内容， 使用 git diff

$ git diff

知道了修改后，把修改后的文件重新添加到仓库

$ git add index.txt

在提交到仓库之前我们再看看仓库当前的状态

$ git status
On branch master
Changes to be committed:
(use "git reset HEAD ..." to unstage)
modified: index.txt

上面告诉我们，将要被提交的修改包括index.txt，下一步，就可以放心地提交了：

$ git commit -m "frist xiugai"
[master 4461b09] frist xiugai
1 file changed, 1 insertion(+)

提交后再看看仓库的状态

$ git status
On branch master
nothing to commit, working tree clean

Git告诉我们当前没有需要提交的修改，而且，工作目录是干净（working tree clean）的。

现在我们已经提交了两个版本了，如果以后提交多个版本肯定记不住，所以用 git log 查看

$ git log
commit 4461b093f72578485c77e5f36f3fe582854eeba2
Author: syf3121 <312191136@qq.com>
Date: Sat May 26 10:07:20 2018 +0800
frist xiugai
commit e226e1d023961b93dacfb4f0b054991964e26c90
Author: syf3121 <312191136@qq.com>
Date: Sat May 26 09:59:26 2018 +0800
frist file

如果要退回那个上个版本或者某个版本，要先知道当前版本，git中HEAD表示当前版本 使用 git reset

$ git reset --hard HEAD^

如果要在回到刚刚的版本，可以在上面git log 里找到那个版本的commit id(只需写前几位就可以)

$ git reset --hard 4461b

如果不小心把git 命令窗口关闭了，可以使用 git reflog

git reflog
4461b09 HEAD@{3}: commit: frist xiugai
e226e1d HEAD@{4}: commit (initial): frist file

也可以看到每一次的commit id

到这，基本常用的命令已经学会，现在我们学习如何与远程仓库关联

一般我们都会申请一个github账号，这样就可以免费获取一个远程仓库

本地git仓库和github仓库之间是通过ssh加密的所以要设置

首先要在自己电脑生生 SSH Key ，在命令行输入 ,注意写成自己的邮箱,然后一路回车就ok

$ ssh-keygen -t rsa -C "312191136@qq.com"

顺利的话可以再用户目录底下 找到 .ssh的目录，里面有id_rsa和id_rsa.pub两个文件， 这两个就是SSH Key的秘钥对，id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人。

然后就可以登录github把ssh密匙添加上，如下图，点击New SSH key添加自己的 id_rsa.pub密匙

到这一步基本就大功告成了，然后去github新建一个仓库，然后和本地仓库关联。这里就不说怎么在github创建仓库了，自行百度
